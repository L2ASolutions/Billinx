# NRS E-Invoicing API Specification
## Billinx Integration Reference Document
### Last updated: May 2026

---

## Overview

The NRS (National Revenue Service) E-Invoicing platform facilitates the secure validation, signing, transmission, and reconciliation of sales invoices between taxpayers and the NRS.

Billinx operates as a **System Integrator (SI)** in this chain:

```
Business (ERP/Dashboard)
    ↓
Billinx (System Integrator)
    ↓
Interswitch or Pillarcraft (Access Point Provider / APP)
    ↓
NRS Platform (Government)
    ↑
IRN + QR Code returned up the chain
```

---

## Authentication

All API requests require two static headers on every call:

```
x-api-key    → Your NRS API key
x-api-secret → Your NRS secret key
```

### Important Notes
- These are static credentials assigned per tenant on registration
- There is NO OAuth token flow — headers must be included on every request
- Keys must be kept confidential and stored encrypted
- Regenerate immediately if compromised
- Always use HTTPS

### Header Format
```http
POST /api/v1/invoice/validate
x-api-key: your-api-key-here
x-api-secret: your-secret-key-here
Content-Type: application/json
```

## IRN Generation

The Invoice Reference Number (IRN) is generated by the System Integrator using three data points:

| Component | Description | Format | Example |
|-----------|-------------|--------|---------|
| Invoice Number | From taxpayer's accounting system | Alphanumeric, no special chars | INV001 |
| Service ID | Assigned by NRS on business registration | 8 alphanumeric characters | 94ND90NR |
| Datestamp | Invoice issue date | YYYYMMDD | 20240611 |

### IRN Format
```
InvoiceNo-ServiceId-YYYYMMDD
```

### Example
```
INV001-94ND90NR-20240611
```

## IRN Signing and QR Code Generation

Every accepted invoice must have a cryptographically signed QR code. This is done using keys downloaded from the NRS dashboard.

### Step 1 — Download Crypto Keys
From NRS dashboard → My Account → Manage Cryptographic Keys → Download `crypto_keys.txt`

The file contains:
```json
{
  "public_key": "LS0tLS1CRUdJTiBQVUJMSUMgS0VZLS0tLS0...",
  "certificate": "bEF0V3FJbmo5cVZYbEdCblB4QVpjMG9HVWFrc29GM2hiYWFkYWMyODRBUT0="
}
```

### Step 2 — Decode Public Key
```bash
jq -r '.public_key' crypto_keys.txt > public_key.txt
openssl base64 -d -in public_key.txt -out public_key.pem
```

### Step 3 — Prepare IRN with Timestamp
Append UNIX timestamp to IRN with period separator:
```
Format: "InvoiceNo-ServiceId-YYYYMMDD.UnixTimestamp"
Example: "INV001-94ND90NR-20240611.1731618237"
```

⚠️ **Critical**: The UNIX timestamp must be appended. The QR code will not verify without it.

### Step 4 — Encrypt Using Public Key
```bash
openssl pkeyutl -encrypt -inkey public_key.pem -pubin -in data.json -out encrypted_data.bin
base64 -i encrypted_data.bin -o encrypted_data.txt
```

### Step 5 — Generate QR Code
Using the `firs-einvoicing` npm package:
```javascript
import { generateQRCode } from "firs-einvoicing";

const result = await generateQRCode({
  irn: "INV001-94ND90NR-20240611.1731618237",  // IRN + UNIX timestamp
  certificate: "bEF0V3FJbmo5...",               // From crypto_keys.txt
  publicKey: "LS0tLS1CRUdJTiBQVUJMSUMgS0VZLS0tLS0...", // From crypto_keys.txt
  size: 300,
  fgColor: "#000000",
  bgColor: "#FFFFFF",
});

// Store these on the invoice record:
const qrCodeBase64 = result.qrCodeDataUrl;
const encryptedIrn = result.encryptedBase64;
```

### Tenant Crypto Key Storage
Each tenant has their own NRS-issued keys. Store on Tenant model:
- `nrsPublicKey` — RSA public key (base64, from crypto_keys.txt)
- `nrsCertificate` — X.509 certificate (base64, from crypto_keys.txt)
- `nrsApiKey` — API key for authentication (encrypted)
- `nrsApiSecret` — Secret key for authentication (encrypted)
- `nrsServiceId` — 8-character service ID for IRN generation
- `nrsBusinessId` — UUID assigned by NRS on registration

## Error Responses

All errors follow this structure:
```json
{
  "errorCode": "400",
  "errorMessage": "Validation Failed",
  "details": "CustomerTIN is required."
}
```

### Common Error Codes
| HTTP | Scenario |
|------|----------|
| 400 | Missing required field, duplicate IRN, invalid business ID |
| 401 | Invalid or missing API key or secret |
| 422 | Schema validation failed |
| 429 | Rate limit exceeded |
| 500 | NRS system error |
| 503 | NRS system unavailable |

## Security Notes

- Store `nrsApiKey` and `nrsApiSecret` encrypted using AES-256-GCM
- Store `nrsPublicKey` and `nrsCertificate` as base64 strings
- Never log API keys or secrets
- Rotate keys immediately if compromised
- All communication must use HTTPS/TLS 1.2+
- QR codes must appear on every printed invoice for NRS compliance
